# 回溯
回溯算法主要应用于搜索。相比于暴力搜索，能排除一些不可能的情况，从而提高效率。

## N 皇后问题
在 $N\times N$ 的网格上放置 $N$ 个皇后，任意两个皇后都不能处于同一行、同一列或同一斜线上。求摆放方式。

枚举每个皇后放置的位置，其中第 $i$ 个皇后需要放在第 $i$ 行。若某一行无法放置，说明先前的摆放方式有问题，可以马上回溯。
## 收费站问题
给定 $n$ 个在 x 轴上的点，其中 $0=x_1<x_2<\dots<x_n$。已知 $n(n-1)/2$ 个距离，请还原这 $n$ 个点的坐标。

设这 $n(n-1)/2$ 个距离中最大的是 $D$，则 $x_n=D$，再将 $D$ 删去，找出剩余距离的最大值，其对应的点只可能是到起点或终点的距离为这个最大值。分别考虑到起点、到终点的情况，检查和已有的所有点的距离是否包含在剩余的距离之中。如果某个距离不存在，说明有问题，回溯。
## 博弈树
博弈树的每一个节点对应一个局面，每一条边对应一个动作。
### 极小极大搜索
对于每个局面，都可以定义一个评价函数。对于每个终止状态可以赋予一个权值。对于其他节点，满足正反方轮流走，正方最大化可能局面的答案，反方最小化可能局面的答案。

由于正反方交替走步，MAX 层和 MIN 层 会交替出现。

其步骤为：

- 构建博弈树
- 将评估函数应用于叶子节点
- 自底向上计算每个节点的 MINMAX 值
- 从根节点选择 MINMAX 值最大的分支，作为行动策略
### alpha-beta 剪枝
对于 MAX 层节点，可以看作评估值的初始值为 $-\infty$，随着子节点的遍历向上增长；而对于 MIN 层节点，可以可以看作评估值的初始值为 $+\infty$，随着子节点的遍历向下减小。

对于 MIN 层节点，上一层会选择评估值最小的节点。因此，可以逐个遍历每个儿子，实时更新 MAX 值与 MIN 值。若某一节点的 MINMAX 值超过了 MIN 值，则最小化答案的人一定不会选择这一分支，因此可以直接剪枝。

对于 MAX 层节点，也同理，若某一节点的 MINMAX 值小于了 MAX 值，则最大化答案的人一定不会选择这一分支，因此可以直接剪枝。
